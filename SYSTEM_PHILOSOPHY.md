# פילוסופיית המערכת — Telegram → MT5

מסמך קצר, ממוקד — לכל צ'אט חדש. **MT5 הוא מקור האמת.**

---

## מבנה המערכת — 3 נתיבים עצמאיים

### 1. ⚡ Fast Path (נתיב הכנסת הפקודות)

**מטרה:** לפתוח עסקאות ב-MT5 בזמן הקצר ביותר האפשרי.

**יעד:** 0.5–1.0 שניות (מקסימום ~1 שניה).

```
Telegram
   ↓
Listener
   ↓
Parser
   ↓
Builder
   ↓
Order Engine
   ↓
MT5
```

**חוקים:**
- זהו **הנתיב הקריטי**.
- **אסור** לבצע בו: Analytics, Recovery, Reports, חישובים כבדים.
- כל פעולה שאינה חובה לפתיחת העסקאות — **רק לאחר** שליחת כל הפקודות.
- **הגדרת קבוצות ואסטרטגיות** (Chat ID → Strategy) נכנסת **לנתיב זה**.

### 2. 🧠 Smart Path (נתיב ניהול, תיעוד וסטטיסטיקה)

**מטרה:** לנהל את **כל מחזור החיים** של הסיגנל — **אחרי** שהעסקאות כבר נפתחו.

**רכיבים:**
- Snapshot Engine
- Signal Workers
- Trade Supervisor
- BreakEven Service
- Safety Service
- Analytics
- Monthly Reports

#### מתועד לכל סיגנל

**פרטי הסיגנל:**
- Provider (שם הקבוצה)
- Chat ID
- Strategy
- Strategy Version
- Signal ID
- Signal UUID

**פרטי כל עסקה:**
- Ticket
- Symbol
- Direction
- Lot
- Entry
- SL
- TP
- Comment
- Magic Number
- Open Time
- Close Time
- Profit

**Timeline (Audit Log):**
- זמן קבלת הודעת הטלגרם
- זמן סיום Parsing
- זמן סיום Builder
- זמן שליחת העסקאות
- זמן פתיחת העסקאות ב-MT5
- זמן הגעה ל-TP3
- זמן קידום Break Even
- זמן סגירת כל עסקה
- זמן סיום הסיגנל

**Execution Metrics** (מחושבים **אחרי** שליחת הפקודות):
- Telegram Received
- Parse Time
- Build Time
- Queue Time
- Order Send Time
- MT5 Execution Time
- Total Execution Time

#### סטטיסטיקות
- מספר סיגנלים לכל קבוצה
- מספר עסקאות שנפתחו
- TP3 / TP4 / TP5 / TP6 Hits
- Original SL Hits
- Break Even SL Hits
- Profit/Loss בדולרים
- דירוג קבוצות לפי רווח

> **כל המדידות מחושבות לאחר שליחת הפקודות** — כדי שלא יעכבו את ה-Fast Path.

---

### 3. 🔄 Recovery System

**מטרה:** לאפשר לבוט להמשיך לעבוד גם לאחר קריסה או הפעלה מחדש.

**עקרונות:**
- **MT5 הוא מקור האמת.**
- Recovery סורק את כל העסקאות של הבוט לפי **Magic Number** ו-**Comment**.
- משחזר Signal Context ו-Trade Context.
- יוצר מחדש **Signal Workers** לכל סיגנל פתוח.
- ממשיך לנהל את העסקאות **כאילו הבוט לא נעצר**.

---

## עקרונות חשובים

1. **MT5 הוא מקור האמת.**
2. **כל סיגנל עצמאי.**
3. **כל Signal** מנוהל ע"י **Signal Worker** משלו.
4. **Trade Supervisor** מנהל את ה-Workers **בלבד**.
5. **כל Feature חדש** חייב להגיע עם **Test**.
6. **כל רכיב** אחראי על **תחום אחד** בלבד.
7. **כל דבר שאינו חובה** לפתיחת העסקאות — **רץ ברקע**.
8. **כל הקבוצות** נשמרות לאורך כל חיי הסיגנל (דוחות והשוואת ביצועים).
9. המערכת נבנית כך שניתן יהיה להוסיף **קבוצות חדשות, אסטרטגיות חדשות ואתר ניתוח נתונים** — **ללא שינוי בליבת המערכת**.

---

## סיכום הפרדה

```
Fast Path     →  קבל → פרס → בנה → שלח  (+ config קבוצות/אסטרטגיות)
Smart Path    →  הכל אחרי השליחה
Recovery      →  שחזור מ-MT5 בעת עלייה
```
